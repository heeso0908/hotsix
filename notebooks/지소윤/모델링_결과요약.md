## 7. Isolation Forest + LightGBM 콜라보 실험

### 실험 배경
LightGBM 단독 모델의 한계를 보완하기 위해 Isolation Forest와의 결합 방법 탐색

### 방법 B: 2단계 파이프라인 (IF → LightGBM)

**구조**
```
공정 데이터 → [IF: 이상 탐지] → 이상 샘플만 → [LightGBM: 불량 분류]
```

**결과 (실패)**

| 지표 | 수치 |
|------|------|
| 이상 탐지 샘플 중 실제 불량 | 0/103 (0.0%) |
| 미탐지 샘플 중 실제 불량 | 115/428 (26.9%) |

- IF가 탐지한 이상 샘플에 실제 불량이 전혀 없음
- 실제 불량 115건이 모두 IF의 "정상" 판정을 받음
- **원인**: 이 데이터의 불량은 공정 수치 분포상 양품과 겹쳐있어 비지도 학습으로 구분 불가

### 방법 1: IF 이상치 점수를 피처로 추가 (decision_function)

**구조**
```
공정 데이터 (21개) + anomaly_score (1개) = 22개 피처 → LightGBM
```

**IF 이상치 점수 분포**

| 클래스 | anomaly_score 평균 |
|--------|-------------------|
| 양품 | 0.0492 |
| 불량 | 0.0658 |

- 불량이 양품보다 점수가 높음 (= IF는 불량을 더 정상으로 판단)
- 두 클래스 간 차이가 매우 작아 유의미한 신호 제공 어려움

**성능 비교**

| 모델 | Recall | F1 | ROC-AUC |
|------|--------|----|---------|
| LightGBM 단독 | 0.800 | **0.533** | **0.789** |
| LightGBM + IF 점수 | **0.843** | 0.527 | 0.781 |

- Recall 소폭 상승, F1/ROC-AUC 소폭 하락
- IF가 의미있는 추가 정보를 제공하지 못함

### 결론
이 데이터에서 IF는 불량 탐지에 직접적인 기여가 어려움. **불량이 공정상 정상처럼 보이기 때문**

## 8. 캐스케이드 파이프라인 (LightGBM → XGBoost)

### 구조
```
전체 데이터
    ↓
[Stage 1: LightGBM - Recall 0.90 목표]
불량 의심 샘플 314건 플래깅 (531건 중)
    ↓
[Stage 2: 튜닝된 XGBoost]
진짜 불량 vs 과탐지 구분
    ↓
최종 불량 판정
```

### 성능 비교

| 모델 | 임계값 | Recall | Precision | F1 | ROC-AUC |
|------|--------|--------|-----------|----|---------|
| LightGBM 단독 | 0.44 | 0.800 | **0.400** | **0.533** | **0.789** |
| Stage 1만 (LGB) | 0.30 | 0.922 | 0.338 | 0.494 | 0.789 |
| **캐스케이드** | 0.47 | **0.904** | 0.342 | 0.496 | 0.775 |

### 핵심 인사이트
- Recall 0.80 → 0.904로 유의미하게 상승
- XGBoost Stage 2가 Precision을 0.338 → 0.342로 소폭 개선
- F1은 단독 LightGBM보다 낮음 (0.533 → 0.496)

### 용도별 최종 모델 선택

| 목적 | 모델 | Recall | F1 |
|------|------|--------|----|
| F1 균형 최우선 | LightGBM 단독 | 0.800 | **0.533** |
| 불량 최대한 탐지 (Recall 최우선) | 캐스케이드 | **0.904** | 0.496 |

## 9. 통계 분석 vs ML(SHAP) 피처 중요도 비교

### 순위 비교표

| 순위 | 통계 (Spearman) | ML (SHAP) | 일치 여부 |
|------|----------------|-----------|---------|
| 1 | Factory_Humidity (-0.278) | Factory_Humidity | ✅ 완전 일치 |
| 2 | Factory_Temp (+0.216) | Coolant_Pressure | ⚠️ 불일치 |
| 3 | Biscuit_Thickness (-0.170) | Factory_Temp | ✅ 일치 |
| 4 | Spray_2_Time (+0.167) | Casting_Pressure | ⚠️ 불일치 |
| 5 | Casting_Pressure (-0.125) | Biscuit_Thickness | ✅ 일치 |
| 6 | Cylinder_Pressure (-0.121) | Air_Pressure | ⚠️ 불일치 |
| 7 | Cycle_Time (-0.120) | Melting_Furnace_Temp | ✅ 일치 |
| 9 | Spray_Time (-0.097) | Cycle_Time | ✅ 일치 |

### 주요 인사이트

**Factory_Humidity — 두 분석 모두 압도적 1위**
- 통계: 효과크기 ε²=0.104 (유일한 "중" 등급)
- ML SHAP: ~0.87로 2위 대비 3배 이상
- 가장 신뢰도 높은 핵심 변수

**Coolant_Pressure — ML은 2위, 통계는 15위**
- 통계 상관계수 -0.042로 매우 낮음
- ML은 다른 변수와의 비선형 상호작용을 포착
- 단순 상관으로는 보이지 않는 패턴 존재 → ML이 추가 발견한 변수

**Spray_2_Time — 통계는 4위, ML은 하위권**
- 단독 상관은 높지만 다른 변수 통제 시 독립 기여 낮음
- 통계가 과대평가한 변수 (다른 변수와 혼재 가능성)

**Cylinder_Pressure — 통계 6위, ML 거의 0**
- Casting_Pressure와 상관계수 0.999 (사실상 동일)
- ML은 다중공선성 자동 처리 → Casting_Pressure만 선택
- 통계의 맹점: 다중공선성 미반영

**Pressure_Diff_ratio — 두 분석 모두 낮음**
- 통계 p=0.975 (유의하지 않음), ML SHAP도 하위권
- 두 분석 모두 동의: 무의미한 변수
