# Isolation Forest 실험 결과 (Product Type 1)

## 실험 목적

LightGBM 단독 모델(Recall 0.80, F1 0.533)의 한계를 보완하기 위해
Isolation Forest와의 결합 방법 두 가지를 탐색

---

## 방법 1. IF 이상치 점수를 피처로 추가

**구조**
```
공정 데이터 (21개) + anomaly_score (1개) = 22개 피처 → LightGBM
```

**IF 이상치 점수 분포**

| 클래스 | anomaly_score 평균 |
|--------|-------------------|
| 양품   | 0.0492            |
| 불량   | 0.0658            |

- 불량이 양품보다 점수가 높음 (IF 기준으로는 불량을 더 "정상"으로 판단)
- 두 클래스 간 차이가 매우 작아 유의미한 신호 제공 어려움

**성능 비교**

| 모델 | 임계값 | Recall | Precision | F1 | ROC-AUC |
|------|--------|--------|-----------|-----|---------|
| LightGBM (단독) | 0.44 | 0.800 | 0.400 | **0.533** | **0.789** |
| LightGBM + IF 점수 | 0.41 | **0.843** | 0.383 | 0.527 | 0.781 |

**소결**: Recall 소폭 상승(+0.043), F1/ROC-AUC 소폭 하락 → IF가 유의미한 추가 정보 제공 못함

---

## 방법 2. 캐스케이드 파이프라인 (LightGBM → XGBoost)

**구조**
```
전체 데이터
    ↓
[Stage 1: LightGBM - Recall 0.90 목표]
불량 의심 샘플 314건 플래깅 (531건 중)
    ↓
[Stage 2: 튜닝된 XGBoost]
진짜 불량 vs 과탐지 구분
    ↓
최종 불량 판정
```

**Stage 1 상세**

| 항목 | 값 |
|------|-----|
| Stage 1 임계값 | 0.30 |
| Stage 1 Recall | 0.922 |
| 플래깅된 테스트 샘플 | 314건 / 531건 |
| Stage 2 학습 데이터 | 1,328건 (양품 872 / 불량 456) |

**성능 비교**

| 모델 | 임계값 | Recall | Precision | F1 | ROC-AUC |
|------|--------|--------|-----------|-----|---------|
| LightGBM (단독) | 0.44 | 0.800 | **0.400** | **0.533** | **0.789** |
| Stage1 LightGBM (Recall 0.90 목표) | 0.30 | 0.922 | 0.338 | 0.494 | 0.789 |
| LightGBM → XGBoost (캐스케이드) | 0.47 | **0.904** | 0.342 | 0.496 | 0.775 |

**소결**: Recall 0.800 → 0.904로 유의미하게 상승, F1은 0.533 → 0.496으로 하락

---

## 종합 결론

| 목적 | 최적 모델 | Recall | F1 |
|------|----------|--------|-----|
| F1 균형 최우선 | LightGBM 단독 | 0.800 | **0.533** |
| 불량 최대한 탐지 (Recall 최우선) | 캐스케이드 | **0.904** | 0.496 |

**IF 콜라보 한계**
- 이 데이터의 불량은 공정 수치 분포상 양품과 겹쳐 있어 비지도 학습으로 구분 불가
- IF 이상치 점수가 불량/양품 클래스를 구별하는 신호로 작동하지 않음
- 캐스케이드는 Recall 향상에 효과적이나, Precision 개선 효과는 제한적
